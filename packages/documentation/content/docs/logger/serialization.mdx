---
title: Advanced Serialization
---

Hive Logger uses advanced serialization to ensure that all attributes are logged safely and
readably, even when they contain complex or circular data structures. This means you can log rich,
nested objects or errors as attributes without worrying about serialization failures or unreadable
logs.

For example, the logger will serialize the error object, including its message and stack, in a safe
and readable way. This advanced serialization is applied automatically to all attributes passed to
log methods, child loggers, and writers.

```ts
import { Logger } from "@graphql-hive/logger";

const log = new Logger();

class DatabaseError extends Error {
  constructor(message: string) {
    super(message);
    this.name = "DatabaseError";
  }
}
const dbErr = new DatabaseError("Connection failed");
const userErr = new Error("Updating user failed", { cause: dbErr });
const errs = new AggregateError([dbErr, userErr], "Failed to update user");

log.error(errs);
```
